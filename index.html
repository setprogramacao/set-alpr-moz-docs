1. Visão Geral do Projeto

Problema Identificado

Na Universidade Jean Piaget de Moçambique, docentes podiam registar a sua presença na portaria e abandonar o campus antes de cumprir a carga horária. Não existia nenhum sistema automático que correlacionasse o momento de chegada e saída dos veículos com o registo de presença.

Solução Proposta

Sistema ALPR (Automatic License Plate Recognition) que detecta automaticamente as placas dos veículos através de câmara IP, usando YOLOv8 + OCR duplo, regista entradas/saídas numa base de dados e aciona a cancela eletrônica via protocolo Modbus TCP.

Objetivos Específicos

  • Detetar e reconhecer placas moçambicanas (formato XXX 000 XX)
  • Registar automaticamente entradas e saídas por placa
  • Controlar cancela eletrônica via PLC (Modbus TCP)
  • Disponibilizar painel web com relatórios e alertas
  • Gestão de utilizadores com controlo de acessos (RBAC)

Formato de Placa — Moçambique

Padrão regex: ^[A-Z]{3}[0-9]{3}[A-Z]{2}$

3.3 Lógica de Alternância Entrada/Saída (Toggle por Placa)

O tipo de movimento (entrada/saída) é determinado automaticamente pelo servidor com base no histórico de cada placa individual — não pelo módulo desktop.

# deteccao_service.py
def determinar_tipo_movimento(db, placa, veiculo_id):
ultimo = obter_ultimo_registro_placa(db, placa)
# Sem histórico → primeira vez = entrada
if not ultimo:
return "entrada"
return "saida" if ultimo.tipo_movimento == "entrada" else "entrada"
Exemplo visual:
1
MZB123MC detectada
Sem histórico → ENTRADA registada
↓ veículo permanece no campus
2
MZB123MC detectada novamente
Último registo = entrada → SAÍDA registada
↓ veículo sai do campus
✓ A lógica é por placa individual — não afetada por outras placas detectadas

Correção Inteligente de Erros OCR — corrigir_placa_ocr()

A função analisa a posição de cada caractere. Posições 0-2 e 6-7 devem ser letras. Posições 3-5 devem ser números. Se o OCR retornou o caractere errado, é corrigido automaticamente:

7. Integração PLC / Cancela Eletrônica

Protocolo Modbus TCP

O Modbus TCP é o protocolo industrial mais utilizado para comunicação com PLCs (Programmable Logic Controllers). Funciona sobre Ethernet/IP, na porta 502 (padrão) / 5020 (simulador).

C0
Coil 0 — Cancela
Saída digital. True = abre cancela. False = fecha.
Função Modbus: FC01 (read) / FC05 (write)
DI0
Discrete Input 0 — Sensor de Laço
Entrada digital. True = veículo no laço indutivo.
Função Modbus: FC02 (read discrete inputs)

Fluxo de Abertura da Cancela

1
Sensor de laço (DI 0 = True) deteta veículo no portão
2
PLCController aciona câmara → OCR processa placa
3
OCR leu placa? → Movimento registado + Coil 0 = True (cancela abre)
✕
OCR falhou? → Cancela NÃO abre → Agente de segurança regista e abre manualmente
4
Sensor OFF (DI 0 = False) → veículo passou → Coil 0 = False (cancela fecha)
⚠
Fallback: se sensor não responder em 8s → cancela fecha por segurança

Simulador PLC Virtual

Para permitir testes sem hardware físico, o sistema inclui um servidor Modbus TCP virtual (desktop_module/simulador_plc.py) que emula o comportamento do PLC e do sensor de laço.

# Iniciar simulador
python -m desktop_module.simulador_plc
Servidor Modbus TCP aguardando em 127.0.0.1:5020
Thread de simulação iniciada (modo AUTO)
Cancela: fechada ↓ | Laço: livre ○
Cancela: ABERTA ↑ | Laço: livre ○
[SIM] Veículo NO laço indutivo (sensor=ON)
[SIM] Veículo SAIU do laço (sensor=OFF)
Cancela: fechada ↓ | Laço: livre ○
Comportamento automático do simulador:
1 Desktop abre cancela (Coil 0 = True)
2 Simulador aguarda 1.5s (simula veículo chegando ao laço)
3 Liga DI 0 = True (veículo no laço)
4 Aguarda 2s (veículo a passar)
5 Liga DI 0 = False → PLCController fecha a cancela
Nota: Em produção, o simulador é substituído pelo PLC real. O desktop_module não precisa de alterações — apenas muda o IP/porta no ficheiro .env.
